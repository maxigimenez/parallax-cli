# Configuration Reference

Parallax stores all configuration in `~/.parallax/config.json`. You do not edit this file manually ‚ÄĒ use `parallax init` or the dashboard to manage it.

## config.json structure

```json
{
  "version": 1,
  "projects": [...],
  "agents": [...],
  "slack": null,
  "secrets": { "LINEAR_API_KEY": "..." },
  "updatedAt": 1716300000000
}
```

### projects

Array of project entries. Each project maps to one repository and one issue source.

#### Minimal project

```json
{
  "id": "my-app",
  "workspaceDir": "/absolute/path/to/repo",
  "pullFrom": {
    "provider": "github",
    "filters": {
      "owner": "myorg",
      "repo": "my-app",
      "state": "open"
    }
  },
  "agent": {
    "provider": "claude-code"
  }
}
```

#### Project fields

**id** (required) ‚ÄĒ unique identifier across all projects, no spaces.

**workspaceDir** (required) ‚ÄĒ absolute path to a local git repository. Must contain `.git/`.

**pullFrom.provider** (required) ‚ÄĒ `github` or `linear`.

**pullFrom.filters** ‚ÄĒ provider-specific:

- GitHub: `owner` (required), `repo` (required), `state`, `labels`
- Linear: `team` (required), `labels`, `state`

**agent.provider** (required) ‚ÄĒ `claude-code`, `codex`, or `gemini`.

**agent.model** (optional) ‚ÄĒ pin a specific model version. Omit to use the provider default.

**agent.name** (optional) ‚ÄĒ reference a named agent from the `agents` array instead of specifying a provider directly.

**agent.systemPrompt** (optional) ‚ÄĒ custom instructions prepended to every prompt for this project.

**agentLabels** (optional) ‚ÄĒ map from issue label to named agent, for per-label agent routing:

```json
{
  "agentLabels": {
    "ai-frontend": "reviewer",
    "ai-security": "reviewer"
  }
}
```

### agents

Array of reusable named agent definitions. Projects reference them by `agent.name`.

```json
{
  "name": "developer",
  "provider": "claude-code",
  "model": "claude-opus-4-5",
  "systemPrompt": "You are a senior backend engineer..."
}
```

Agent names must be unique. `provider` is required; `model` and `systemPrompt` are optional.

### slack

Slack bot configuration, or `null` if not configured.

```json
{
  "botToken": "xoxb-...",
  "appToken": "xapp-...",
  "channel": "#eng-ai"
}
```

Managed from the **Integrations ‚Üí Slack** tab in the dashboard or during `parallax init`.

See [Slack Bot](./slack-bot.md) for the full setup guide.

### secrets

Key-value map of environment variables injected into the orchestrator process at startup. Agent processes inherit them automatically.

```json
{
  "LINEAR_API_KEY": "lin_api_...",
  "SOME_OTHER_KEY": "value"
}
```

Managed from the **Secrets** tab in the dashboard. Values are masked in the UI (`‚ÄĘ‚ÄĘ‚ÄĘ‚ÄĘ‚ÄĘ‚ÄĘ‚ÄĘ`) and never returned by the API.

Common secrets:

- `LINEAR_API_KEY` ‚ÄĒ required if any project uses Linear as the issue provider

## Managing configuration

| Where | What you can do |
|---|---|
| `parallax init` | First-time setup wizard; add a project |
| Dashboard ‚Üí Projects | Add, edit, delete projects |
| Dashboard ‚Üí Integrations | Configure GitHub, Linear, Slack |
| Dashboard ‚Üí Secrets | Add, update, delete secrets |

Changes made in the dashboard take effect immediately without restarting Parallax.
